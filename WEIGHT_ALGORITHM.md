# Fortune Fusion Engine - 权重算法详解

## 概述

Fortune Fusion Engine 采用**动态加权融合算法**，将10大命理体系的计算结果通过多维权重加权合并为一个综合评分。

核心公式：

```
动态权重 = 基础权重 × 场景适配系数 × 历史准确率 × 置信度 × 用户校准系数
综合评分 = Σ(各体系评分 × 动态权重) / Σ(动态权重)
```

---

## 1. 基础权重（Base Weights）

10大命理体系的全局基础权重，总和为 **0.60**（归一化由引擎自动处理）：

| 体系 | 基础权重 | 说明 |
|------|---------|------|
| 八字命理 | 0.12 (20%) | 最高权重，千年验证体系 |
| 紫微斗数 | 0.10 (16.7%) | 命盘格局分析核心 |
| 七政四余 | 0.08 (13.3%) | 中国传统占星，天文基础 |
| 奇门遁甲 | 0.08 (13.3%) | 时空预测，事件导向 |
| 大六壬 | 0.06 (10%) | 古代三式之一，精细推断 |
| 西方占星术 | 0.06 (10%) | 心理分析与趋势预测 |
| 六爻纳甲 | 0.04 (6.7%) | 具体事件占卜 |
| 梅花易数 | 0.02 (3.3%) | 灵活快速，直觉导向 |
| 数字命理学 | 0.02 (3.3%) | 数字振动分析 |
| 姓名学 | 0.01 (1.7%) | 辅助参考 |

**设计原理**：权重分配基于三个维度——历史验证深度、理论体系完整度、实际预测准确率。

## 3. 动态权重计算

每个体系的最终权重由以下5个因子相乘得到：

```
动态权重 = 基础权重 × 场景适配 × 历史准确率 × 置信度 × 用户校准
```

### 各因子说明

| 因子 | 取值范围 | 默认值 | 说明 |
|------|---------|--------|------|
| 基础权重 | 0.01~0.12 | 见上表 | 体系固有权重 |
| 场景适配 | 0.5~1.5 | 1.0 | 场景对体系的适配程度 |
| 历史准确率 | 0.0~1.0 | 0.8 | 该体系的历史预测准确率 |
| 置信度 | 0.0~1.0 | 计算得出 | 本次计算的确定性 |
| 用户校准 | 0.5~2.0 | 1.0 | 用户个性化调整 |

### 计算示例

以"具体事件"场景下，奇门遁甲的动态权重计算：

```
基础权重 = 0.08
场景适配 = 1.5（具体事件场景对奇门遁甲的系数）
历史准确率 = 0.8（默认值）
置信度 = 0.85（本次计算的置信度）
用户校准 = 1.0（默认值）

动态权重 = 0.08 × 1.5 × 0.8 × 0.85 × 1.0 = 0.0816
```

## 4. 综合评分计算

```python
# 伪代码
for each system in results:
    dynamic_weight[system] = base × scene × accuracy × confidence × user_calibration
    weighted_score[system] = score[system] × dynamic_weight[system]

final_score = sum(weighted_score) / sum(dynamic_weight)
```

**关键特性**：
- 权重自动归一化，不需要手动调和为1
- 置信度低的体系自动降权
- 场景不匹配的体系自动降权

---

## 5. 冲突检测

当两个体系的评分差距 ≥ 50 分时，标记为冲突：

| 分差 | 严重度 | 处理方式 |
|------|--------|---------|
| 50~69 | medium | 共识优先（consensus_priority） |
| ≥70 | high | 人工复核（manual_review） |

冲突不影响最终评分，但会在结果中标注，提醒用户结果存在分歧。

---

## 6. 不确定性量化

```
uncertainty = 标准差(所有体系评分)
```

- **uncertainty < 10**：体系间高度一致，结果可靠
- **uncertainty 10~20**：存在分歧，结果中等可靠
- **uncertainty > 20**：体系间严重分歧，结果需谨慎参考

---

## 7. 可靠性评估

| 条件 | 可靠性 |
|------|--------|
| uncertainty < 10 且 高置信度体系占比 ≥ 60% | **high** |
| uncertainty < 20 且 高置信度体系占比 ≥ 40% | **medium** |
| 其他情况 | **low** |

---

## 8. 趋势判定

| 综合评分 | 趋势 | 含义 |
|---------|------|------|
| ≥ 70 | positive | 吉，整体向好 |
| 50~69 | neutral | 平，稳中待变 |
| < 50 | negative | 凶，需谨慎应对 |

---

## 9. 风险等级

各体系的风险等级（none/low/medium/high/critical）映射为数值（0~4），加权平均后映射回等级：

| 平均风险值 | 风险等级 |
|-----------|---------|
| < 0.5 | none |
| 0.5~1.5 | low |
| 1.5~2.5 | medium |
| 2.5~3.5 | high |
| > 3.5 | critical |

---

## 10. 用户个性化校准

支持为每个用户维护独立的权重校准系数：

```python
user_calibration = {
    "weights": {
        "八字命理": 1.2,    # 该用户八字预测准确率高，加权
        "梅花易数": 0.8,    # 该用户梅花易数准确率低，降权
    }
}
```

通过用户反馈（1~5分）持续优化个性化校准系数。
